CODE-LEVEL FINDINGS
================================================================================

Issue Location: packages/core/src/apps/file-explorer.ts (lines 316-324)

Broken Code:
┌─────────────────────────────────────────────────────────────┐
│ buildViewModel(state, ctx) {                                │
│   return {                                                  │
│     currentPlace: state.currentPlace,     // ← Tracked      │
│     files: getOrderedFiles(...)           // ← Not filtered │
│   };                                                        │
│ }                                                           │
└─────────────────────────────────────────────────────────────┘

The currentPlace is available in the state but is not used to filter files.

Contributing Factors:
  1. FileEntry type has no location field (types.ts)
  2. getOrderedFiles() has no filtering logic (filesystem.ts)
  3. buildViewModel() doesn't use currentPlace for filtering (file-explorer.ts)

Result: Three separate systems that need to be integrated but aren't.

================================================================================
HOW TO FIX IT (QUICK REFERENCE)
================================================================================

Three Required Changes:

1. Add Location Metadata (types.ts)
   Add: location?: "Home" | "Desktop" | "Documents" | "Downloads" | "workspace"
   To: FileEntry type

2. Implement Filtering (file-explorer.ts lines 316-324)
   Change: files: getOrderedFiles(...)
   To:     files: getOrderedFiles(...).filter(f => 
             !f.location || f.location === state.currentPlace)

3. Assign Locations (starter-tasks.ts)
   When creating files, specify their location parameter

These three changes would complete the feature.
