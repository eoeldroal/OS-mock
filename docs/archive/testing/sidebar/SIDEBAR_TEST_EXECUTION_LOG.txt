================================================================================
ROOT CAUSE ANALYSIS
================================================================================

Why Files Don't Change When Selecting Different Places:

The File Explorer sidebar was designed with two disconnected subsystems:

System 1: UI State Management
  - Tracks currentPlace (Home, Desktop, Documents, Downloads, workspace)
  - Updates correctly when sidebar items are clicked
  - Status: WORKING ✓

System 2: File System Display
  - Returns all files without filtering
  - Has no awareness of places/locations
  - Does not check currentPlace before displaying files
  - Status: INCOMPLETE ✗

The Two Systems Are NOT Connected:
  - System 1 says "user selected Home"
  - System 2 shows "all files"
  - Result: User sees no change despite state change

================================================================================
WHAT WOULD FIX THIS
================================================================================

Three Changes Required:

Change 1: Add location to FileEntry type
  File: packages/core/src/types.ts
  Add field: location?: "Home" | "Desktop" | ... (optional)

Change 2: Filter files in buildViewModel
  File: packages/core/src/apps/file-explorer.ts
  In buildViewModel: Filter files by currentPlace before returning

Change 3: Assign locations when creating files
  File: packages/core/src/tasks/starter-tasks.ts
  When calling createFile(): pass location parameter

These three changes would complete the feature and make file filtering work.
